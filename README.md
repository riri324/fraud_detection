# Synthetic Data Generation using CTGAN

This repository contains the code, experiments, and documentation for my 
final year project on synthetic data generation using CTGAN and FinDiff, 
applied to credit card fraud detection.

## Research Question

Can synthetic data generated by CTGAN and FinDiff improve fraud detection 
performance on imbalanced datasets? Specifically, does forcing a higher 
fraud ratio during synthetic data generation lead to better classification 
outcomes compared to preserving the natural class distribution?

- **default+of+credit+card+clients** Taiwan Default of Credit Card Clients 
  Dataset. Used to evaluate the generalisability of the approach on a 
  different tabular dataset.

## Custom Implementation

To enable fraud ratio control during synthetic data generation, the 
original CTGAN library was forked and modified to accept a `ratio` 
parameter. This allows forced oversampling of the minority class during 
generation.

Custom CTGAN fork: https://github.com/riri324/CTGAN

When `ratio=None`, the model generates data preserving the natural class 
distribution. When a float is provided (e.g. `ratio=0.3`), the model 
attempts to force the specified fraud ratio in the generated output.
